# 🖥️ نظام المبيعات والحاسبة الذكي والمحاسبة الذكي (Smart POS & Accounting System)

هو نظام بيع ومحاسبة وإدارة مبيعات متكامل، تم تصميمه ببرمجة **React + TypeScript + Express** مع تصميم مذهل لأسلوب **أجهزة الزجاج الناعم (Soft Glass Neumorphism)**. يدعم العمل الكامل بدون انترنت واجهة رشيقة ثنائية اللغة (عربي/إنجليزي) ويدعم فواتير الطباعة الحرارية وتصدير Excel.

---

## ✨ المميزات الأساسية (Core Features)

1. **🔒 شاشة الدخول والأمان (Access Control & Roles):**
   - مستويان من الصلاحيات: مدير النظام (**Admin**) وأمين الصندوق (**Cashier**).
   - الكاشير يُمنع تماماً من تفتيش التقارير المالية والتحليلات أو تعديل وحذف السجلات الحساسة والمخازن.
   - بيانات تسجيل الدخول الافتراضية:
     - **Admin (كامل الصلاحيات):** اسم مستخدم `admin` وكلمة مرور `password`.
     - **Cashier (صلاحيات محدودة):** اسم مستخدم `cashier` وكلمة مرور `1234`.

2. **📊 شاشة لوحة المعلومات الرئيسية (Dashboard Panel):**
   - بطاقات إحصائية للمبيعات اليومية، مبيعات الشهر، وصافي الأرباح المقدرة (تُحتسب في وقتها الفعلي عبر خصم سعر التكلفة الفعلي للوحدة من سعر البيع!).
   - رسم بياني تفاعلي جذاب باستخدام **Recharts** للمقارنة البصرية بين المبيعات وصافي الأبح المكتسبة للـ 7 أيام الأخيرة.
   - رادار تنبيهي للمخزون المنخفض والمنتجات التي أوشكت على النفاد.

3. **🛒 شاشة نقطة البيع الذكية (Smart POS Console):**
   - فلترة ميسرة للمنتجات حسب الأقسام وبحث ذكي حي.
   - **محاكي مدمج لقارئ الباركود (Barcode Handheld Scanner Gun Emulator):** يمكنك كتابة الرمز أو مسحه يدويًا ليقوم النظام بإضافته فورًا لسلة الشراء تلقائيًا.
   - خصومات تلقائية وضريبة القيمة المضافة ومسترجع الفكة النقدي التلقائي.
   - واجهة دفع مرنة: نقدي، شبكة مدى، أو شغل بالآجل مع ربطه بملف عميل معلق.
   - **فاتورة افتراضية حرارية تفاعلية:** بمجرد الدفع، تُعرض فاتورة تحاكي الفاتورة الحرارية 80 مم مع خيار طباعة حقيقية أو استخراج ملف **PDF مباشر عبر jsPDF**.

4. **📦 إدارة وجدولة المخازن (Inventory & Product Management):**
   - إضافة وتعديل الفئات، التكاليف، أسعار البيع، والحدود الدنيا لتنبيه النفاد.
   - أدوات متكاملة **لاستيراد وتصدير المخزون عبر ملفات CSV (Excel)** بضغطة زر واحدة!

5. **👥 ملفات العملاء والموردين (Directory Registry):**
   - إضافة وتعديل وحساب الأرصدة المعلقة والديون (آجل) وتصدير السجلات إكسل.

6. **💸 إدارة المصاريف والتشغيل (Daily Shop Expenses Control):**
   - رصد الفواتير، الإيجارات، الرواتب، وتكاليف النثرية مع حساب مجموعها وتصنيفها مباشرة.

7. **📈 التقارير المالية والتحليلات (Strict Financial Audits):**
   - فلترة تاريخية متطورة وفلترة بموظف الكاشير مستخرج الفواتير.
   - استخراج تقارير PDF مبوبة ومنسقة وتصدير السجلات كاملة إلكترونيا.

8. **⚙️ الإعدادات العامة للمتجر (Global Customizations):**
   - تغيير لغة النظام فورياً لجميع النواتج والطباعة وطريقة العرض.
   - تحديد العملة المتداولة (SAR, USD, EGP)، وتعيين قيمة الضريبة والطباعة الافتراضية (A4 أو طباعة إيصالات حرارية).

## 🎨 متطلبات التصميم (Soft Glass Neumorphism visual theme)

- **خلفيات زجاجية شفافة:** دمج رائع بين تأثير الضباب والبلورة الخلفية (`backdrop-filter`) مع حدود فضية ناعمة.
- **أزرار neumorphic ثلاثية الأبعاد:** ظلال مزدوجة دافئة (ظل داكن خارجي + ظل لامع خلفي) تتجاوب وتنغلق ديناميكياً عند النقر والتحويم.
- **تأثيرات حركة Framer Motion:** انتقالات رشيقة وناعمة للغاية تضفي طابعاً برمجياً غاية في الفخامة والاحترافية.

## 🛠️ كيف يعمل النظام بدون اتصال بالشركة (Offline-First Dual-Sync Design)

تم بناء هيكلية تخزين ذكية ثنائية الطبقة:
1. يحاول النظام الحفظ واسترجاع السجلات عبر خادم الـ **Full-Stack Express API** المتصل بقاعدة بيانات `database.json` التلقائية التأسيس.
2. في حال سحب الخادم محلياً أو تشغيل التطبيق في بيئة ثابتة (Static Web Browser File)، يقوم النظام تلقائياً وبأجزاء من الثانية بالتحول إلى محرك تخزين **localStorage** المحلي الداخلي للمتصفح.
3. تظل كل لوحات التحكم ونقط المبيعات والمخازن والعملاء والمصروفات تعمل وتُبنى بشكل متكامل وقوي بنسبة **100% offline** وبدون تأثر إطلاقاً بوجود اتصال إنترنت!

## ⚙️ أوامر تشغيل النظام محلياً (How to Run Locally)

لتشغيل هذا النظام على جهاز الكمبيوتر الخاص بك، اتبع الأوامر البسيطة التالية:

```bash
# 1. تثبيت الحزم التبعية المطلوبة
npm install

# 2. تشغيل واجهة وخادم الـ Fullstack Express + Vite
npm run dev

# 3. بناء المشروع والتجميع النهائي للإنتاج
npm run build

# 4. تشغيل التطبيق كبرنامج سطح مكتب محلي مستقل عبر Electron
npm run electron
```

```text
│   │   ├── Customers.tsx    # تجمعات ديون ومستحقات العملاء
│   │   ├── Suppliers.tsx    # بيانات الشركات الموردة المعتمدة
│   │   ├── Expenses.tsx     # قيود المصاريف التشغيلية
│   │   ├── Reports.tsx      # ملفات تحصيل الضرائب والأرباح الصافية
│   │   ├── Settings.tsx     # تكوين الضرائب، مسميات المتجر، وحجم الطباعة
│   │   └── Users.tsx        # إدارة حسابات موظفي المبيعات والامتياز الكاشير
│   ├── /contexts
│   │   └── AppContext.tsx   # القلب النابض للنظام وحافظ الحالة والمحركات ثنائية المزامنة

│   │   └── translations.ts  # قاموس الترجمة الكامل الفوري اللغوي المعرب والمنسق بالكامل
│   ├── tsconfig.json
│   ├── index.css            # فضاء Cairo و Inter وتوليفات CSS Neumorphic الزجاجية
│   └── main.tsx
├── server.ts                # الخادم المتكامل والـ Express REST API
├── database.json            # ملف قاعدة البيانات المشكلة ذاتياً عند خط التشغيل الأول
└── electron.js              # معزز تشغيل الواجهة بصورة برنامج كفء مستقل
```
